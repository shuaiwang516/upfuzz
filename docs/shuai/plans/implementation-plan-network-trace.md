# Implementation Plan: Network Trace Guided Fuzzing

## Summary of Current State

From our verified experiment (Cassandra 3.11.17 → 4.1.4), the current traces look like:

```
TraceEntry{methodName='SEND_GOSSIP_DIGEST_SYN', hashcode=-64349562,
           changedMessage=false, recentExecPath=[0,0,0,...,0], log='null'}
```

Three problems:
1. **DEF is dead** — `recentExecPath` is all zeros because `Runtime.hit()` is never called
2. **changedMessage is always false** — `modifiedFields.json` not loaded into Docker containers
3. **Hashcode ignores DEF and MSG details** — `hashcode = name.hashCode()` only; `recentExecPathHash` and `changedMessage` are stored but never used in Jaccard similarity

This plan fixes all three, then connects the similarity to the fuzzing feedback loop.

Modified code:
```java
if (Utils.isComparisonCondition(expr)) {
    int currentId = dumpIdCounter++;

    // NEW: Insert Runtime.hit() BEFORE the if-statement
    String hitStmtStr = Utils.constructHitStmt(currentId);
    Statement hitStatement = StaticJavaParser.parseStatement(hitStmtStr);
    int idx = newStatements.indexOf(stmt);
```

## Phase 3: Enrich the Hashcode (Make Similarity Use DEF + MSG)

**Goal:** Jaccard similarity considers execution path and message content, not just verb name.

**Effort:** Small — change one line in Trace.java and one in TraceEntry.java.

### Step 3.1: Change hashcode computation in Trace.record()

**Repo: ssg-runtime**
**File:** `src/main/java/org/zlab/net/tracker/Trace.java` (line 32)

Current:
```java
traceEntries.add(new TraceEntry(id, name, name.hashCode(), changedMessage,
        System.currentTimeMillis(), recentExecPathHash, recentExecPath, payloadType));
```

Changed — combine verb name, execution path hash, and changedMessage flag:
```java
int enrichedHashcode = java.util.Objects.hash(
    name.hashCode(),
    recentExecPathHash,
    changedMessage
);
traceEntries.add(new TraceEntry(id, name, enrichedHashcode, changedMessage,
        System.currentTimeMillis(), recentExecPathHash, recentExecPath, payloadType));
```

### Step 3.2: ~~Verify getHashCodes() uses the enriched hashcode~~ [SUPERSEDED]

**[SUPERSEDED]** `getHashCodes()` and `DiffComputeJaccardSimilarity` have been removed.
Current comparison uses `Trace.getCanonicalMultiset()` with `DiffComputeSemanticSimilarity`.

### Step 3.3: Impact on Jaccard similarity

After this change:
- Two `SEND_MUTATION` entries with **different execution paths** will have **different hashcodes** → lower similarity → fuzzer notices
- A `SEND_MUTATION` carrying a **changed message** will have a **different hashcode** than one without → fuzzer prioritizes it
- The 2-gram generation will capture these differences in sequence context

```java
public static void recordPost() {
    if (trace == null) return;
    int[] postPath = snapshot();
    synchronized (lock) {
        trace.attachPostExecPath(postPath);
    }
}
```

### Step 5.3: Add `attachPostExecPath()` to Trace

**Repo: ssg-runtime**
**File:** `src/main/java/org/zlab/net/tracker/Trace.java`

```java
public void attachPostExecPath(int[] postPath) {
    // Find the most recent RECV entry and attach post-execution path
    for (int i = traceEntries.size() - 1; i >= 0; i--) {
        TraceEntry entry = traceEntries.get(i);
        if (entry.methodName.startsWith("RECV_") && entry.postExecPath == null) {
            entry.postExecPath = postPath;
            entry.postExecPathHash = Utils.computeHash(postPath);
            break;
        }
    }
}
```
